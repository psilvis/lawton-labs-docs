Graduel

Politique de confidentialité

Dernière mise à jour : 26 février 2025

1. Aperçu

Graduel (« l'Application ») est conçue selon une approche axée sur la confidentialité. Vos données de santé sont stockées de manière sécurisée sur votre appareil et synchronisées via le service iCloud d'Apple utilisant CloudKit. Vos données ne sont jamais stockées sur les serveurs de Lawton Labs. Certaines fonctionnalités font appel à des services tiers, comme décrit ci-dessous.

2. Données que nous collectons

L'Application collecte et stocke les données suivantes localement sur votre appareil :

  • Informations de profil — Votre nom et vos préférences saisis lors de la configuration initiale.
  • Données de santé — Données provenant d'Apple HealthKit (pas, fréquence cardiaque, VFC, sommeil, VO2max et autres indicateurs) avec votre autorisation explicite.
  • Données d'activité — Informations sur les entraînements et activités provenant de Strava.
  • Résultats d'analyses — Valeurs de biomarqueurs extraites des rapports d'analyses que vous importez.
  • Protocoles de suppléments — Suppléments, dosages et journaux d'observance que vous créez.
  • Bilans d'humeur — Évaluations quotidiennes de l'humeur et du stress que vous enregistrez.

Toutes les données ci-dessus sont chiffrées sur l'appareil et synchronisées de manière sécurisée via Apple iCloud (CloudKit). Vos données sont stockées dans votre compte iCloud personnel et ne sont jamais stockées sur les serveurs de Lawton Labs.

3. Données partagées avec des tiers

OpenAI (Traitement des rapports d'analyses)

Lorsque vous utilisez la fonctionnalité d'importation d'analyses et choisissez l'analyse par IA, votre rapport d'analyses au format PDF est envoyé à l'API d'OpenAI pour l'extraction automatisée des biomarqueurs. Cela signifie :

  • Votre rapport d'analyses PDF — qui peut inclure votre nom, les valeurs de biomarqueurs, les plages de référence, le nom du laboratoire et les dates des tests — est transmis aux serveurs d'OpenAI pour traitement.
  • OpenAI traite ces données conformément aux Conditions d'utilisation de l'API et à la Politique de confidentialité d'OpenAI.
  • Conformément à la politique d'utilisation des données de l'API d'OpenAI, les données envoyées via l'API ne sont pas utilisées pour entraîner les modèles d'OpenAI.
  • Les données sont transmises via une connexion HTTPS chiffrée.
  • Aucun identifiant d'appareil ni information personnelle au-delà du contenu du rapport n'est transmis.
  • Si l'analyse par IA échoue, l'Application utilise un analyseur local basé sur des expressions régulières qui ne transmet aucune donnée.

Vous pouvez éviter le partage de vos données d'analyses avec OpenAI en n'utilisant pas la fonctionnalité d'importation PDF. L'option « Coller les résultats » utilise un analyseur embarqué qui ne transmet aucune donnée.

4. Données Apple HealthKit

Les données de santé accédées via Apple HealthKit sont traitées conformément aux directives d'Apple pour HealthKit :

  • Les données ne sont accédées qu'après votre autorisation explicite.
  • Les données HealthKit sont uniquement lues — l'Application n'écrit pas de données dans HealthKit.
  • Les données HealthKit sont synchronisées via votre compte iCloud personnel à l'aide de CloudKit d'Apple. Ces données sont stockées dans votre conteneur iCloud privé et ne sont jamais stockées sur les serveurs de Lawton Labs ni transmises à des tiers.
  • Les données HealthKit ne sont jamais utilisées à des fins publicitaires ou marketing.

5. Intégration Strava

Lorsque vous connectez votre compte Strava :

  • L'authentification utilise OAuth 2.0 avec PKCE — l'Application ne voit ni ne stocke jamais votre mot de passe Strava.
  • Seules les données d'activité (type, durée, distance, fréquence cardiaque) sont récupérées via l'API de Strava.
  • Les données d'activité récupérées sont mises en cache localement sur votre appareil.
  • Vous pouvez déconnecter Strava à tout moment depuis l'écran Profil.

6. Stockage et sécurité des données

  • Les données sensibles (clés API, jetons, données de santé) sont stockées à l'aide d'expo-secure-store, qui utilise le trousseau iOS (Keychain) pour un stockage chiffré.
  • Les données de santé stockées dans SQLite sont chiffrées avec AES-256-GCM au niveau applicatif. La clé de chiffrement est stockée dans le trousseau iOS (Keychain).
  • Vos données sont synchronisées vers votre compte iCloud personnel via CloudKit d'Apple. Ces données sont stockées dans un conteneur CloudKit privé accessible uniquement par vous. Aucune donnée de santé n'est stockée sur les serveurs exploités par Lawton Labs.
  • L'exportation des données est disponible au format JSON depuis l'écran Profil.

7. Conservation et suppression des données

  • Toutes les données sont stockées sur votre appareil et synchronisées vers votre compte iCloud personnel. Les données persistent jusqu'à ce que vous les supprimiez.
  • Vous pouvez supprimer toutes vos données à tout moment en utilisant l'option « Supprimer toutes les données » sur l'écran Profil. Cela supprime les données de votre appareil et d'iCloud.
  • La désinstallation de l'Application supprime toutes les données stockées localement. Les données de votre compte iCloud peuvent persister jusqu'à leur suppression via l'Application ou les réglages iCloud.
  • Les données envoyées à OpenAI pour l'analyse des rapports sont traitées de manière transitoire et ne sont pas conservées par OpenAI à des fins d'entraînement de modèles, conformément aux politiques d'utilisation des données de l'API d'OpenAI.

8. Confidentialité des enfants

L'Application n'est pas destinée aux personnes de moins de 13 ans. Nous ne collectons pas sciemment d'informations personnelles auprès d'enfants de moins de 13 ans.

9. Analyse et suivi des erreurs

L'Application utilise les services suivants pour l'analyse et le suivi des erreurs :

  • Amplitude — Analyse anonyme de l'utilisation pour nous aider à comprendre comment les fonctionnalités sont utilisées et à améliorer l'Application. Aucune donnée de santé ni information personnelle identifiable n'est envoyée à Amplitude.
  • Sentry — Rapports de plantages et d'erreurs pour nous aider à identifier et corriger les bugs. Les rapports d'erreurs peuvent inclure le type d'appareil, la version du système d'exploitation et les traces d'exécution, mais n'incluent pas vos données de santé.

Aucun de ces services ne reçoit vos données de santé, biomarqueurs ou informations de santé personnellement identifiables.

10. Modifications de cette politique

Nous pouvons mettre à jour cette Politique de confidentialité de temps à autre. Nous vous informerons des modifications importantes via l'Application. L'utilisation continuée de l'Application après les modifications constitue l'acceptation de la politique mise à jour.

11. Vos droits

Vous avez le droit de :

  • Accéder à toutes vos données stockées dans l'Application.
  • Exporter vos données dans un format portable (JSON).
  • Supprimer toutes vos données à tout moment.
  • Déconnecter tout service tiers à tout moment.
  • Choisir de ne pas utiliser les fonctionnalités impliquant un traitement de données par des tiers (importation d'analyses).

12. Contact
